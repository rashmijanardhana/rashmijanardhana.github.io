Variables and literals are the building blocks of any Python program. Understanding how Python stores and represents data is the first step to writing clean, effective code.

## Variables

**Variable** — A named reference to a value stored in memory. In Python, you don't declare a type — just assign and go.

```python
name = "Alice"
age = 30
is_active = True
```

**Dynamic Typing** — Python figures out the type at runtime. The same variable can hold different types at different points, though that's usually a bad idea.

```python
x = 10       # int
x = "hello"  # now a str — valid, but avoid this
```

**Multiple Assignment** — Python lets you assign multiple variables in one line.

```python
a, b, c = 1, 2, 3
x = y = z = 0  # all point to the same value
```

**Naming Rules** — Variable names must start with a letter or underscore, can contain letters, digits, and underscores, and are case-sensitive. `myVar` and `myvar` are different.

## Literals

**Integer Literal** — A whole number with no decimal point. Python supports different bases too.

```python
decimal = 100
binary  = 0b1010   # 10 in decimal
octal   = 0o17     # 15 in decimal
hex_val = 0xFF     # 255 in decimal
```

**Float Literal** — A number with a decimal point or in scientific notation.

```python
pi = 3.14159
tiny = 1.5e-3   # 0.0015
```

**String Literal** — Text enclosed in single, double, or triple quotes.

```python
greeting = "Hello, World!"
multiline = """This spans
multiple lines."""
raw = r"C:\Users\name"   # raw string, backslashes not escaped
```

**Boolean Literal** — Only two values: `True` and `False`. Note the capital first letter.

```python
is_valid = True
has_error = False
```

**None Literal** — Represents the absence of a value. Python's equivalent of null.

```python
result = None
```

## Type Checking

**type()** — Use this to inspect what type a variable currently holds.

```python
print(type(42))        # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>
print(type(None))      # <class 'NoneType'>
```

## Quick Reference

| Literal Type | Example         | Python Type  |
|--------------|-----------------|--------------|
| Integer      | `42`            | `int`        |
| Float        | `3.14`          | `float`      |
| String       | `"hello"`       | `str`        |
| Boolean      | `True`          | `bool`       |
| None         | `None`          | `NoneType`   |

> Variables are just labels. Python manages the actual objects in memory — two variables can point to the same object without you realizing it. Use `is` vs `==` carefully.
