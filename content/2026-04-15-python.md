FastAPI is one of the fastest ways to build APIs in Python. On Day 1, we cover the core concepts — what an API is, how GET and POST work, and how to test them using Swagger UI.

## What is an API?

Think of an API like a waiter in a restaurant. You (the app) tell the waiter what you want, the waiter goes to the kitchen (server/database), and brings back your order (data). The API is the messenger in between.

**API** — Application Programming Interface. It connects two systems so they can talk to each other.

## What is FastAPI?

**FastAPI** — A modern Python web framework for building APIs quickly. It is fast, easy to read, and comes with automatic documentation built in.

```python
from fastapi import FastAPI

app = FastAPI()
```

Run it with:

```bash
uvicorn app:app --reload
```

## GET Request

**GET** — Used to *fetch* or *retrieve* data from the server. You are not sending any new data — just asking for something back.

```python
@app.get("/student/{name}")
def get_student(name: str):
    return {"student": name}
```

- The `{name}` in the URL is a **path parameter**
- Visit `http://127.0.0.1:8000/student/Rashmi` to get a response

## POST Request

**POST** — Used to *send* data to the server. You are submitting something new — like adding a student record.

```python
@app.post("/student")
def add_student(student: StudentModel):
    return {"message": "Student added", "data": student}
```

## Pydantic Models

**Pydantic** — Defines the shape and type of data you send in a POST request. FastAPI uses it to validate incoming data automatically.

```python
from pydantic import BaseModel

class StudentModel(BaseModel):
    name: str
    age: int
```

If you send the wrong type (e.g., age as a string), FastAPI rejects it with a clear error — no extra code needed.

## How Data Flows in a POST Request

1. You send a POST request with JSON body `{"name": "Rashmi", "age": 21}`
2. FastAPI receives it and validates against `StudentModel`
3. If valid, your function runs and processes the data
4. A response is sent back

## How GET Retrieves Data

1. You call `GET /student/Rashmi`
2. FastAPI matches the route and extracts `name = "Rashmi"`
3. Your function runs and returns the data
4. Response comes back as JSON

## Swagger UI

FastAPI auto-generates interactive API docs at:

```
http://127.0.0.1:8000/docs
```

**Swagger UI** — A browser-based tool to test all your endpoints without needing Postman or curl. You can send GET and POST requests directly from the browser and see live responses.

## Quick Reference

| Method | Purpose        | Data Location  | Example URL         |
|--------|----------------|----------------|---------------------|
| GET    | Fetch data     | URL / params   | `/student/Rashmi`   |
| POST   | Send new data  | Request body   | `/student`          |

> FastAPI validates your data, generates docs, and handles errors — all out of the box. That's why it's a top choice for building GenAI backends.
